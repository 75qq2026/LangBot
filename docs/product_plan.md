# AI 智能客服 & 企业助手平台 — 产品规划

## 产品定位

整合 **LangBot**（多平台 IM 机器人开发平台）+ **WeChat-AIChatbot-WinOnly**（企微/微信个人号 Hook），打造一站式 AI 智能客服与企业助手平台。

**大目标**：SaaS 模式，多租户按需订阅
**短期目标**：定制化部署 + Agent 调教
**初级版本**：独立部署，微信客服 + 资料收集 + 通用 Agent

## 三个版本功能规划

### V1.0 基础版 — 独立部署通用 Agent

> **目标用户**：中小企业、个体商户
> **部署方式**：私有化部署（一键 Docker 部署）
> **核心价值**：开箱即用的 AI 客服 + 客户资料收集

| 功能模块 | 功能点 | 适用场景 |
|---------|--------|---------|
| **消息通道** | 微信客服（wecomcs）— 外部客户通过微信直接咨询 | 外部客户对话 |
| | 企微智能机器人（wecombot）— 内部单聊 + 群聊 | 企业内部使用 |
| | 企微个人号 Hook（ntwork）— 企微个人号消息收发 | 外部群、1对1 |
| | 微信个人号 Hook（ntchat）— 个人微信消息收发 | 外部群、1对1 |
| **AI 对话** | 通义千问 qwen-plus（联网搜索已开启） | 全场景 |
| | 可切换 GPT-4 / DeepSeek / Claude 等主流模型 | 全场景 |
| | 多轮对话上下文记忆 | 全场景 |
| | 流式回复（打字机效果） | 企微智能机器人 |
| **客户资料收集** | 自动记录对话历史（文本 + 图片） | 1对1 客户对话 |
| | 图片接收与识别（OCR / 多模态理解） | 资料收集 |
| | 自动提取客户信息（姓名、电话、需求等） | CRM 对接 |
| | 对话标签 & 分类 | 客户管理 |
| **基础管理** | Web 管理后台（LangBot UI） | 运维管理 |
| | 机器人启停、管道配置 | 运维管理 |
| | 对话记录查看 & 导出 | 数据分析 |
| | System Prompt 自定义（人设调教） | Agent 定制 |

**V1.0 覆盖的使用场景：**

| 场景 | 通道 | 能力 |
|------|------|------|
| 企业内部员工提问 | 企微智能机器人 | AI 问答、知识库查询 |
| 内部群组 AI 助手 | 企微智能机器人 @机器人 | 群内 AI 回复 |
| 外部客户微信咨询 | 微信客服 | AI 自动接待、资料收集 |
| 外部群 AI 客服 | 企微个人号 Hook / 微信客服 | 群内 AI 回复、客户服务 |
| 1对1 客户跟进 | 企微/微信个人号 Hook | 自动回复、信息采集 |

> **核心价值**：行业专属 Agent + 多系统打通 + 精细化运营

| 功能模块 | 功能点 | 适用场景 |

| | 多知识库绑定（不同群/客服 → 不同知识库） | 多产品线 |
| | Agent 工作流编排（引导式对话收集信息） | 表单收集、预约 |
| | 意图识别 + 自动路由（转人工/转专属客服） | 智能分流 |
| **多模态能力** | 图片识别 → 结构化数据提取 | 证件采集、单据识别 |
| | 图片生成（产品图、海报） | 营销 |
| | 语音消息识别 + 语音回复 | 语音交互 |
| | 文件接收与解析（PDF/Word/Excel） | 资料处理 |
| **系统对接** | API 网关（Webhook 推送对话事件） | 自定义开发 |
| | CRM 对接（自动创建客户、更新跟进记录） | 销售管理 |
| | ERP/OA 对接（查库存、查订单、提工单） | 企业内部 |

| | 敏感词过滤 & 合规审计 | 风控合规 |

| **客户资料管理** | 结构化客户档案（从对话自动提取） | CRM |

## 技术选型建议：LangBot 二开 vs 芋道框架

### 方案对比

| 维度 | 方案 A：LangBot 二开 | 方案 B：芋道 (yudao) + LangBot | 推荐 |
|------|---------------------|-------------------------------|------|
| **V1.0 基础版** | ✅ 直接用，零额外开发 | ❌ 过度设计 | **LangBot** |
| **V2.0 专业版** | ✅ LangBot 已有 RAG/MCP/插件体系 | ⚠️ 芋道管后台更成熟，但 AI 能力需自建 | **LangBot 为主** |
| **V3.0 SaaS 版** | ⚠️ 缺多租户/计费/权限体系 | ✅ 芋道天然支持多租户/RBAC/计费 | **芋道 + LangBot** |
| AI/IM 能力 | ✅ 核心优势 | ❌ 无 | LangBot |
| 多租户 | ❌ 需自建 | ✅ 内置 | 芋道 |
| 权限管理 | 基础（单用户） | ✅ RBAC 完整 | 芋道 |
| 计费系统 | ❌ 无 | ✅ 支付/订单/套餐 | 芋道 |
| 工作流引擎 | Pipeline（AI 专用） | ✅ Flowable/Activiti | 芋道 |
| 前端 | Next.js + shadcn | Vue3 + Element Plus | 各有优势 |
| 开发语言 | Python (后端) + TS (前端) | Java (后端) + Vue (前端) | 看团队 |

### 推荐策略：分阶段演进

```
V1.0 ──────────────────► V2.0 ──────────────────► V3.0
纯 LangBot                LangBot + 轻量扩展        芋道(管控层) + LangBot(AI层)
零额外开发                  二开 Pipeline/插件         微服务拆分
```

**V1.0 → V2.0**：直接在 LangBot 上二开
- LangBot 已经有 RAG 知识库、MCP 工具调用、插件系统
- 加客户资料管理模块（扩展数据库 + API）
- 加 Agent 工作流编排（扩展 Pipeline 配置）
- 投入小，见效快

**V2.0 → V3.0**：引入芋道作为管控层
- 芋道负责：多租户、权限、计费、审计、SSO
- LangBot 作为 AI 引擎服务，通过 API 被芋道调用
- 架构：芋道(Java) → API Gateway → LangBot(Python)
- 每个租户对应一组 LangBot 实例/配置

### 结论

> **V1.0 和 V2.0 不需要芋道，直接 LangBot 二开最高效。**
> **V3.0 做 SaaS 时再引入芋道框架，负责商业化管控层。**
> 不要过早引入复杂框架，先用最小成本把产品跑通、验证市场。
